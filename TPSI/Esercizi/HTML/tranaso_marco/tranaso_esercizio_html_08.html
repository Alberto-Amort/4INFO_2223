host locale

Nelle reti di computer, localhost è un nome host che si riferisce al computer corrente utilizzato per accedervi . Viene utilizzato per accedere ai servizi di rete in esecuzione sull'host tramite l'interfaccia di rete di loopback. L'uso dell'interfaccia di loopback ignora qualsiasi hardware dell'interfaccia di rete locale.

Ritorno

Il meccanismo di loopback locale può essere utilizzato per eseguire senza richiedere un'interfaccia di rete fisica o senza rendere il servizio accessibile dalle reti a cui il computerpuò essere connesso. Ad esempio, è possibile accedere a un sito Web installato localmente da un browser Web tramite l'URL http://localhost per visualizzare la sua home page. Il nome localhost normalmente si risolve nell'indirizzo di loopback IPv4 127.0.0.1 e nell'indirizzo di loopback IPv6 ::1.

Risoluzione del nome

Gli standard di rete IPv4 riservano l'intero blocco di indirizzi 127.0.0.0/8 (più di 16 milioni di indirizzi) per scopi di loopback.[2] Ciò significa che qualsiasi pacchetto inviato a uno di quegli indirizzi viene eseguito in loop. L'indirizzo 127.0.0.1 è l'indirizzo standard per il traffico di loopback IPv4; il resto non è supportato da tutti i sistemi operativi. Tuttavia, possono essere utilizzati per configurare più applicazioni server sull'host, tutte in ascolto sullo stesso numero di porta. Lo standard IPv6 assegna un solo indirizzo per il loopback: ::1. La risoluzione del nome localhost su uno o più indirizzi IP è normalmente configurata dalle seguenti righe nel file hosts del sistema operativo: 127.0.0.1 host locale ::1 host locale Il nome può essere risolto anche dai server DNS (Domain Name System), ma le query per questo nome devono essere risolte localmente e non devono essere inoltrate a server dei nomi remoti. Oltre alla mappatura di localhost agli indirizzi di loopback (127.0.0.1 e ::1), localhost può anche essere mappato ad altri indirizzi IPv4 (loopback) ed è anche possibile assegnare altri o ulteriori nomi a qualsiasi indirizzo di loopback . Non è garantito che la mappatura di localhost a indirizzi diversi dall'intervallo di indirizzi di loopback designato nel file hosts o nel DNS abbia l'effetto desiderato, poiché le applicazioni potrebbero mappare il nome internamente. Nel DOmain Name System, il nome localhost è riservato come nome di dominio di primo livello, originariamente messo da parte per evitare confusione con il nome host utilizzato per scopi di loopback.[3] Gli standard IETF vietano ai registrar di nomi di dominio di assegnare il nome localhost..

standard IETF


Il nome localhost è riservato a scopi di loopback da RFC 6761 (Nomi di dominio a uso speciale),[4] che ha raggiunto il livello di maturità Proposed Standard nel febbraio 2013. Lo standard stabilisce una serie di considerazioni speciali che regolano l'uso del nome nel Domain Name System: Una query di indirizzo IPv4 o IPv6 per il nome deve sempre essere risolta nel rispettivo indirizzo di loopback, specificato in uno standard separato. Le applicazioni possono risolvere il nome in un indirizzo di loopback da sole o passarlo ai meccanismi di risoluzione dei nomi locali . Quando un risolutore di nomi riceve una query di indirizzo (A o AAAA) per localhost, deve restituire gli indirizzi di loopback appropriati e risposte negative per qualsiasi altro tipo di record richiesto. Le query per localhost non devono essere inviate ai server dei nomi di memorizzazione nella cache. Per evitare di sovraccaricare di traffico i server root del Domain Name System, i server dei nomi di memorizzazione nella cache non devono mai richiedere record del server dei nomi per localhost o inoltrare la risoluzione ai server dei nomi autorevoli. I registrar DNS non possono delegare nomi di dominio nel localhost di dominio di primo livello . Quando i server dei nomi autorevoli ricevono query per 'localhost' nonostante le disposizioni menzionate, risolverle in modo appropriato. Gli indirizzi di loopback IPv4 sono riservati all'interno dello spazio degli indirizzi IPv4 dallo standard IETF "Special Use IPv4 Addresses" (RFC 5735).[5] La prenotazione può essere fatta risalire allo standard "Numeri assegnati" del novembre 1986 (RFC 990). Al contrario, lo standard IETF "IP Version 6 Addressing Architecture" (RFC 4291) riserva l'unico indirizzo di loopback IPv6 ::1 all'interno dello spazio degli indirizzi IPv6. Lo standard preclude l'assegnazione di tale indirizzo a qualsiasi interfaccia fisica, nonché il suo utilizzo come indirizzo di origine o destinazione in qualsiasi pacchetto inviato a host remoti. Casi speciali

Le versioni del database MySQL distinguono tra l'uso del nome host localhost e l'uso degli indirizzi 127.0.0.1 e ::1.[7] Quando si utilizza localhost come destinazione nell'interfaccia di un connettore client di un'applicazione, l'interfaccia di programmazione dell'applicazione MySQL si connette al database utilizzando un socket di dominio Unix, mentre una connessione TCP tramite l'interfaccia di loopback richiede l'uso diretto dell'indirizzo esplicito.
